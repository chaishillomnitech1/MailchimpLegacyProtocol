# MailchimpLegacyProtocol

This repository serves as a central hub for integrating Mailchimp workflows into various marketing strategies. It provides workflows, documentation, and resources to enhance collaboration and innovation in marketing automation.

---

## Kingdom Directive Framework

The **Kingdom Directive** is a perpetual operational framework embedded in this repository. It defines the guiding principles, workflows, and automation patterns that ensure every contribution and integration remains aligned with the repository's mission of continuous improvement and innovation.

### Core Principles

- **Alignment** – Every workflow and automation is designed to align with the repository's marketing mission and the evolving needs of its collaborators.
- **Adaptability** – Modular templates and reusable scripts ensure components can be extended or repurposed across future integrations.
- **Perpetual Evolution** – Continuous learning loops are encoded into the framework so that knowledge and tooling evolve in real time with operational requirements.
- **Transparency** – All directives, workflows, and API hooks are documented openly so collaborators can understand, audit, and contribute.

### Quick Start

1. **Review AGENTS.md** for operational guidelines before contributing.
2. **Explore `docs/`** for the full Kingdom Directive specification and API hook reference.
3. **Use templates in `templates/`** as starting points for new campaigns or audience segments.
4. **Run scripts in `scripts/`** to integrate with Mailchimp's API or synchronize the knowledge base.

### Workflows

| Workflow | Description |
|---|---|
| `campaign-automation.yml` | Automates Mailchimp email campaign scheduling and dispatch |
| `subscriber-sync.yml` | Keeps subscriber lists synchronized with external data sources |
